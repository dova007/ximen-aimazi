# 文风对比分析指南

> **何时加载**：用户选择文风 F（对比学习文风）时，Phase 4 阶段加载本文件作为分析方法论。
>
> **作用**：定义从两段对比样本（原文 vs 修改文）中提取可执行写作规则的标准化流程。10 个维度覆盖网文写作最关键的风格差异点，确保分析有据可依、规则可落地。

---

## 概述

本指南的核心思路与 `style-fingerprint-guide.md` 一致：将风格定性判断转化为可量化的维度对比。不同的是，风格指纹比对是「续写是否偏离原作」，而本指南是「修改文比原文好在哪，如何固化这些好」。

---

## 十项分析维度

### 维度 1：开篇方式

**分析要点**：
- 第一段是什么？环境描写 / 对话 / 动作 / 心理活动 / 设定说明
- 读者打开这一章的前 3 秒看到了什么？

**原文常见 AI 模式**：
- 以天气/环境描写开篇（"窗外的风声凄厉至极"）
- 先铺场景，再引入角色

**修改文常见优化方向**：
- 以对话冲突直接切入（"'滚开！……'"）
- 以角色正在做的事开篇，场景信息嵌入动作中

**规则提取模板**：
```
规则：开篇必须用 [动作/对话/冲突] 切入，严禁以 [环境描写/心理活动/设定说明] 起头。
禁止：开头超过 [N] 句仍未出现人物动作或对话。
```

---

### 维度 2：句式长度

**计算方法**（参照 `style-fingerprint-guide.md`）：
- 以句号、问号、感叹号为分隔切分句子
- 计算平均句长和句长分布（短≤10字 / 中 11-25字 / 长>25字）

**对比要点**：
- 修改文的平均句长是否明显短于原文？
- 修改文是否增加了短句比例、减少了超长句？

**规则提取模板**：
```
规则：平均句长控制在 [N] 字以内，短句占比不低于 [X]%。
禁止：连续 [N] 句超过 [M] 字。
```

---

### 维度 3：段落结构

**计算方法**：
- 以空行为分隔切分段落
- 统计每段行数、段落间节奏切换频率

**对比要点**：
- 修改文段落是否更短、更碎？
- 修改文是否避免了大段连续叙述块？

**AI 味特征**：
- 连续 5 行以上的叙述块
- 段落间缺少对话或动作打断

**规则提取模板**：
```
规则：段落不超过 [N] 行，动作/对话/叙述交替出现，每 [M] 段切换一次节奏。
禁止：连续 [N] 段纯叙述（无对话、无动作）。
```

---

### 维度 4：对话密度

**计算方法**：
- 识别引号标记的对话行
- 对话行数 / 总行数

**对比要点**：
- 修改文的对话占比是否更高？
- 修改文是否更多用对话推进情节而非叙述？

**规则提取模板**：
```
规则：对话占比不低于 [X]%，关键剧情推进优先用对话而非旁白叙述。
禁止：连续 [N] 段只有叙述没有对话。
```

---

### 维度 5：情绪表达

**分析要点**：
- 原文的情绪表达方式：直接心理描写 vs 动作暗示
- 修改文的情绪表达方式：有何变化？

**AI 味特征（需要警惕的）**：
- "他感到一阵……"、"他心中涌起……"
- "他忍不住……"、"他不由得……"
- "一股……涌上心头"
- 直接用情绪词描述角色内心状态

**优化方向（需要模仿的）**：
- 通过动作暗示："他垂下眼帘，看着泥泞中散落的黑色药渣，眼神深处闪过一抹阴郁"
- 通过对话或沉默表达
- 让读者自己感受到情绪，而非被告知

**规则提取模板**：
```
规则：情绪表达优先通过 [动作/对话/环境映射] 暗示，减少直接心理描写。
禁止：[具体的 AI 味情绪表达句式列表]
```

---

### 维度 6：描写风格

**分析要点**：
- 形容词密度：每百字形容词数量
- 修饰性描写 vs 功能性描写的比例
- 感官描写的分布（视觉/听觉/嗅觉/触觉）

**AI 味特征**：
- 堆砌形容词和副词（"凄厉至极"、"纷纷扬扬"、"破烂的"、"霉烂味道愈发显得刺鼻"）
- 为描写而描写，不服务于剧情推进
- "仿佛"、"犹如"、"像是" 等比喻堆砌

**优化方向**：
- 描写服务于剧情和冲突
- 精准的功能性描写，去掉冗余修饰
- 不写与当前冲突无关的景物

**规则提取模板**：
```
规则：描写必须服务于 [剧情推进/气氛营造/人物塑造]，删减不推进故事的纯粹景物描写。
禁止：[形容词密度超过 X]/[使用比喻词 Y]/[不服务剧情的环境描写]
```

---

### 维度 7：金手指/设定揭示

**分析要点**：
- 系统/金手指是怎么揭示给读者的？
- 是仪式感满满的"觉醒"，还是轻描淡写地嵌入日常？

**AI 味特征（需避免的）**：
- "嗡！脑海深处炸开一声巨响"
- "半透明面板闪烁着微光轰然铺展"
- "一行金色小字缓缓浮现"
- 系统第一次出现被写成重大事件

**优化方向**：
- 金手指早已存在，只是现在才提及
- 用一句话带过："金手指终于到账了"
- 功能面板只在角色实际使用时才展示

**规则提取模板**：
```
规则：金手指/系统揭示必须 [平淡化/嵌入日常/一句话带过]，严禁戏剧化渲染。
禁止：[嗡/炸开/光芒/轰然/铺展] 等仪式感描写用于系统首次出现。
```

---

### 维度 8：节奏控制

**分析要点**：
- 原文叙事是线性还是非线性？
- 修改文是否使用了时间跳跃、信息压缩？
- 场景切换频率

**AI 味特征**：
- 严格按照时间顺序："醒来→观察环境→发现穿越→回忆→触发系统"
- 不敢跳跃时间，把过渡段落写得很满

**优化方向**：
- 直接切入时间点，省去"之前发生了什么"的铺垫
- 信息通过当前冲突侧面透露，而非线性展开
- 敢于留白

**规则提取模板**：
```
规则：叙事节奏以冲突驱动，敢于 [时间跳跃/信息压缩/留白]，不按线性时间事无巨细展开。
禁止：穿越/重生类型必须写"醒来发现穿越"的过渡段落。
```

---

### 维度 9：禁用词汇

**对比方法**：
- 提取原文中高频出现、但修改文中被删除或替换的词/句式
- 特别关注那些典型的 AI 生成文本特征词

**常见 AI 味词汇清单（在对比中确认）**：
| 类别 | 常见表达 | 问题 |
|------|---------|------|
| 时间过渡 | 此刻、就在这时、忽然之间、在这一刻 | 冗余过渡词 |
| 感官渲染 | 仿佛、犹如、好似、如同 | 比喻堆砌 |
| 程度修饰 | 越来越……、愈发、愈发显得、极为 | 形容词通胀 |
| 情绪直述 | 心中涌起、忍不住、不由得、一股……涌上心头 | 说教式情绪 |
| 身体细节 | 缓缓、轻轻、慢慢、微微 | 万能缓动词 |
| 眼神描写 | 眼中闪过、眼眸中透出、目光中带着 | AI 眼神三件套 |
| 感知句式 | 他感到、他意识到、他发现、他察觉 | 过滤式叙述 |

**规则提取模板**：
```
规则：禁用以下词汇/句式：[具体列表]。
规则：出现以下词汇时强制替换：[替换对照表]。
```

---

### 维度 10：人称/叙述距离

**分析要点**：
- 第三人称叙述者离角色有多近？
- 全知视角干预程度：上帝视角解释 vs 限制在角色感知范围内

**AI 味特征**：
- 上帝视角插入："在这个妖魔横行，以强横气血定尊卑的武道世界里……"
- 叙述者对角色内心状态的直接报告
- "原身"、"这具身体" 等穿越者视角的说明性用词

**优化方向**：
- 贴紧视角角色，只写角色能感知到的
- 背景设定通过角色遇到的具体冲突露出
- 减少叙述者的跳脱评价

**规则提取模板**：
```
规则：第三人称近距离，只写视角角色可感知的信息，设定通过冲突自然露出。
禁止：上帝视角的设定说明段落、叙述者跳脱评价。
```

## 对比分析执行流程

### Step 1：加载样本
读取用户提供的原文样本和修改文样本。

### Step 2：逐维度对比
按照上述 10 个维度，逐一对比分析两段文本：
- 原文在该维度的表现（反例）
- 修改文在该维度的表现（正例）
- 差异程度（轻微/中等/显著）

### Step 3：提取规则
将差异转化为可执行的写作规则，每条规则包含：
- **规则编号**：C1-C10，对应维度编号
- **规则说明**：一句话描述规则内容
- **原文反例**：从原文样本中摘录的具体反例
- **修改文正例**：从修改文样本中摘录的具体正例
- **禁止事项**：明确的禁止清单

### Step 4：输出规则集
将提取的规则写入 `memory/style_contrast_guide.md`，格式见该文件模板。

## 与风格指纹比对的关系

| 维度 | 风格指纹指南 | 对比分析指南 |
|------|------------|------------|
| 用途 | 续写一致性 | 风格偏好固化 |
| 基线来源 | 原作已写章节 | 用户提供的对比样本 |
| 输出 | 9项指标偏离度报告 | N条可执行写作规则 |
| 阈值 | ±15-25% 定量 | 定性差异 + 反例/正例 |
| 阻断 | 3项超阈值阻断润色 | 规则强制执行，不设阻断 |

两者可以叠加使用：项目同时激活对比学习文风 + 续写模式时，风格指纹比对继续执行，但基线替换为修改文的指纹数据。
