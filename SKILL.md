## 核心哲学

1. **大纲是地图，不是牢笼** — 粗到能看见全局，细到能坐下就写
2. **爽点密度决定存亡** — 每3000-5000字必须有一个让读者"爽"的情绪节点
3. **日更是底线，质量是上限** — 稳定产出比单章完美更重要
4. **先写30章，再谈其他的** — 大纲不需要完美，写出来自然发现方向
5. **网文写作是工程，不是灵感** — 靠工程可以写200万字，靠灵感不行

Phase 10 正文生成前必须通过：①大纲存在 ②当前批次细纲已冻结 ③无未解决逻辑矛盾。任一不满足→拒绝进入正文。

### 项目进度检测（续写场景）

| 检测标志 | 判断结果 | 恢复位置 |
|---------|---------|---------|

## Phase 1：灵感生成

**目标**：确定小说基本方向，生成3套完整创意方案

### 提示词自动完善（8维度）

按以下维度自动补全用户输入：①题材定位 ②世界观设定 ③主角人设 ④核心冲突 ⑤爽点设计 ⑥节奏规划 ⑦配角框架 ⑧开篇钩子

### 快速问卷（可跳过）

如用户方向模糊：①题材 ②核心卖点 ③目标字数 ④对标书

### 对标上下文加载

如用户提到对标书或 `对标/` 目录存在：检查 `拆文报告.md`，有则读取，无则提示是否先做拆文分析。

### 输出

1. **3个备选书名** 2. **一句话简介**（主角+困境+金手指+目标） 3. **核心冲突** 4. **核心爽点**

## Phase 2：世界观构建

**目标**：基于选定题材构建完整、自洽的世界观

确定4要素：①时代背景 ②力量体系 ③社会结构 ④核心规则

设计要点：力量体系须有边界和代价 / 世界观展开跟剧情推进绑定 / 日常细节比宏大设定更让世界"活" / 每5章须有世界观新信息

> 详细世界观构建模板见 `references/plot-structures.md` 和 `assets/STYLE-TEMPLATE.md`。

### 输出文件

- `设定/世界观/背景设定.md`：时代背景与核心规则
- `设定/世界观/力量体系.md`：力量等级与边界
- `设定/题材定位.md`：题材定位与核心卖点

## Phase 3：角色塑造

**目标**：创建立体、有深度的角色

### 主角卡

```
姓名 / 身份标签 / 外貌特征(3-5关键词) / 性格关键词(3-5个，须有矛盾面)
核心目标 / 核心动机(情感驱动) / 致命弱点 / 金手指/系统(能力+限制) / 口头禅/标志动作
```

### 配角卡

```
姓名 / 角色功能(导师/盟友/情报源/牺牲品/镜像对照) / 与主角关系
核心特质(1-2关键词) / 标志性特征(一句话) / 退场方式
```

### 反派层级

| 层级 | 跨度 | 设计要点 |
|------|------|---------|
| 小反派 | 1-5章 | 1-2个鲜明特征，出场快退场干脆 |
| 中等反派 | 10-30章 | 有动机+有手段+有逼格，至少赢主角一次 |
| 大弧Boss | 一卷或数卷 | 完整人物弧线，有理念冲突 |
| 最终Boss | 全书 | 从第一章就有伏笔，代表主题反面 |

**反派铁律**：反派的智商/实力决定主角的含金量。

> 完整角色设计（角色功能定位+动机链+人物关系）见 `references/author-style-guide.md` 和 `references/examples.md`。

### 输出文件

- `.learnings/CHARACTERS.md`：角色档案（主角、配角、反派）
- `设定/关系.md`：角色关系网络

## Phase 4：创作参数配置（交互式）

**目标**：在生成大纲前，让用户确认关键创作参数

### 模式选择

用户可指定创作模式，未指定时默认标准模式：

| 模式 | 适用场景 | 流程 |
|------|---------|------|
| 🚀 快速模式 | 只想快速生成一章试试 | 跳过 Phase 1-3 详细设定，直接进入 Phase 6-10 |
| 📖 标准模式（默认） | 正常创作流程 | 完整 Phase 1-12 |
| 🔬 深度模式 | 精细打磨大纲+编辑部流水线 | 完整 Phase 1-12 + 编辑部流水线 + 深度审稿 |

### 必问参数（使用 `question` 工具询问）

**渐进式询问**：先问最关键的参数，其他参数在需要时再问。如用户有明确想法可直接给出，如无则提供智能推荐。

```
参数1 章节数量：50章(试水)/100章(标准)/200章(长篇)/500章(超长篇)/自定义/智能推荐
参数2 女角色数量：1个(唯一女主)/2个(双女主)/3个(经典)/5个(后宫)/自定义/智能推荐
参数3 擦边等级：无(纯剧情)/少量(5-10%)/中量(15-25%)/大量(30-40%)
```

**智能推荐**：根据题材自动建议参数。如用户选择"智能推荐"，按以下规则推荐：
- 玄幻/修仙长篇 → 200章+，2-3个女角色，少量擦边
- 都市爽文 → 100章，1-2个女角色，中量擦边
- 悬疑/惊悚 → 50章，0-1个女角色，无擦边
- 轻松日常 → 100章，2-3个女角色，无擦边

如选择擦边（非"无"）→ 询问擦边描写风格：A含蓄/B直白/C文艺/D暴力美学/E自定义

### 输出文件

- `设定/创作参数.md`：章节数/女角色数/擦边等级/文风选择（**必须创建**，Phase 6 前置依赖）
- `memory/project_style.md`：项目级文风圣经

### 5种预设文风

> 完整文风预设库见 `assets/STYLE-TEMPLATE.md`，作家技法参考见 `references/author-style-guide.md`。

## Phase 5：同人作品原著资料（如适用）

**目标**：同人作品收集原著设定，确保创作一致性

**非同人** → 跳过。**同人** → 选择添加方式：搜索添加（询问原著名称→搜索→整理→保存 `设定/原著资料.md`）或粘贴添加（提示用户粘贴→整理→保存）

同人创作约束：世界观须符合原著 / 原著角色性格不能崩坏 / 不能超出原著能力体系 / 专有名词必须用原著术语

## Phase 6：大纲生成

**目标**：制定完整的小说结构

### 前置检查

- `设定/题材定位.md` — 缺失→需先完成 Phase 1-2
- `设定/创作参数.md` — 缺失→需先完成 Phase 4

### 大纲创建五步法

1. **确定高潮剧情** — 最大冲突规模、最多出场人物、最强情绪波动
2. **确定单元剧** — 金手指获得爽点，相邻单元剧不能用相同逻辑
3. **故事线组织** — 同时管理8条线（地图/阵营/人物/金手指/世界观/矛盾/收集/感情线）
4. **确定开篇舞台** — 情节钩子→异常→金手指激活→矛盾→目标
5. **确定收尾** — 解决主线+额外奖励+角色结局+下卷铺垫

### 关键节点（100章基准，按比例缩放）

| 节点 | 比例 | 内容 |
|------|------|------|
| 开篇钩子 | 1% | 必须吸引读者 |
| 第一个高潮 | 10% | 确立故事方向 |
| 中点转折 | 50% | 提升故事格局 |
| 至暗时刻 | 65% | 情感最低点 |
| 最终高潮 | 85% | 所有矛盾爆发 |

> 完整大纲技法（三幕式+升级感设计+矛盾设计+卷级大纲模板）见 `references/plot-structures.md` 和 `references/chapter-outline.md`。

### 输出文件

- `大纲/大纲.md`：全书卷级结构
- `大纲/卷纲_第X卷.md`：每卷爽点节奏+情绪弧线+伏笔+反转

## Phase 7：细纲规划

**目标**：把全书细纲改造成可迭代的批次工作流

### 前置检查

- `大纲/大纲.md` — 缺失→需先完成 Phase 6

### 规划原则

1. 先卷级→后批次→再到单章 2. 默认只做当前批次+下一批缓冲 3. 按风险调整批次大小 4. 区分硬锚点与软空间 5. 先定义冻结条件再开始细纲

### 推荐批次大小

| 场景 | 建议每批章数 |
|------|-------------|
| 开篇前20章 | 3-5章 |
| 稳定推进期 | 5-8章 |
| 大高潮/换地图/换卷 | 2-4章 |

> 详细批次规划规则、冻结条件和模板见 `references/chapter-outline.md`。

### 输出文件

- `大纲/细纲.md`：细纲主控文件（含卷级规划与批次规划表）
- `.learnings/SUSPENSE.md`：伏笔状态表（**必须创建**，Phase 8 前置依赖）

## Phase 8：生成细纲

**目标**：只生成当前批次的详细细纲，为后续返修预留接口

### 前置检查

- `大纲/细纲.md` — 缺失→需先完成 Phase 7
- `.learnings/SUSPENSE.md` — 缺失→Phase 7 输出

### 生成规则

1. 只生成当前批次 2. 带着上下文生成（至少读取前一个已冻结批次+当前卷目标） 3. 每章必须有功能 4. 给返修留接口 5. 批次生成完成后先不写正文，必须先过Phase 9审核并冻结

### 批次细纲内容

每章细纲包含：章节定位、兑现内容、情节点(3-5个)、人物变化、世界变化、伏笔管理、章末钩子、返修接口

### 输出文件

- `大纲/细纲.md`（主控版）
- `大纲/批次细纲/第001-005章_v1.md`（当前批次草稿）

## Phase 9：逻辑审核与人工介入

**目标**：审核当前批次，给出可选方案，让用户介入后决定冻结/小修/回退

### 审核维度

| 维度 | 检查项 |
|------|--------|
| 时间线 | 章节衔接、时间跳跃、成长速度 |
| 空间线 | 地点转移、行动距离、地图切换成本 |
| 人物线 | 存活状态、等级匹配、关系变化、动机连贯 |
| 设定线 | 力量体系、物品来源、社会规则、系统边界 |
| 伏笔线 | 埋设、回收时机、后续可承接性 |
| 读感线 | 章节功能、爽点兑现、章末钩子有效性 |

### 审核流程

1. 读取上一批已冻结细纲+当前批次细纲+当前卷目标 2. 逐章+跨章节检查 3. 记录问题并判断级别（微调/重做/回改） 4. 生成2-4个方案+自定义选项，进入人工介入 5. 用户选择后修改并重新审核，直到满足冻结条件

### 冻结条件

无硬逻辑矛盾 / 本批次承诺的爽点已兑现 / 本批次结尾能自然牵引下一批 / 人工介入节点已完成

## Phase 10：正文章节

**目标**：按审核通过的细纲生成高质量正文

### 前置门控

①`大纲/大纲.md` 存在 ②当前批次细纲已冻结 ③无未解决的逻辑矛盾。缺失时提示：`"无法进入正文：{缺失项}。请先完成 Phase {N}。"`

### 写作前读取

1. `正文/第{N-1}章_*.md`（上一章正文） 2. `大纲/批次细纲/`（当前批次细纲） 3. `.learnings/SUSPENSE.md`（待回收伏笔） 4. `设定/角色/{相关角色}.md`（本章涉及角色） 5. `设定/题材定位.md`（核心梗与对标参考）

### 开头设计

黄金一章法则：从故事最精彩、最有冲突的地方写起。三种开局模板（困境切入/优势切入/悬念切入）详见 `references/plot-structures.md` 中的经典爽文结构公式。

### 钩子技法

> 完整钩子设计详见 `references/plot-structures.md` 中的章节衔接技巧和 `references/chapter-outline.md` 中的章末钩子设计。

高频速查：章尾→突然揭示/紧急危机/未完成动作/身份反转/留白钩子；章首→悬念对话/闪前碎片/倒计时

### 写作技法与节奏

> 写作技法速查和字数节奏参考详见 `references/plot-structures.md` 中的单章节奏公式和 `references/examples.md` 中的完整示例。

### 去AI味写作规范（正文必遵）

> 详见 `references/anti-ai-detection.md`

核心规则：①段落不超过4行 ②动作+对话+情绪反应循环推进 ③短句优先 ④口语化表达 ⑤Show Don't Tell

修辞按题材分级使用：热血爽文风限制使用（仅功能性比喻），古风仙侠风适度放宽（每章≤2处意境比喻），都市现实风/悬疑惊悚风严格限制。擦边风格C（文艺唯美型）允许在擦边情节中使用比喻。

禁用词高频项（频率限制）：仿佛/宛如/一丝/一抹→每章≤1处；深吸一口气/不禁/微微/轻轻→每1000字≤2个；眼中闪过/嘴角勾起→每章≤1次；心中一动/不由得→用动作展示

章末禁止：❌ 总结性感悟/升华式感叹/哲理式收尾/伏笔式预告 → ✅ 用动作、对话或悬念收束

### 输出文件

- `正文/第XXX章_章名.md`

### 记忆写入（正文生成后必执行）

**必写**：`.learnings/PLOT_POINTS.md`（关键情节）、`.learnings/SUSPENSE.md`（伏笔）、`output/CHAPTERS.md`（章节索引）、`SESSION.md`（会话状态）

**有变化则写**：`.learnings/CHARACTERS.md`（角色变化）、`.learnings/LOCATIONS.md`（新地点）、`.learnings/RESOURCES.md`（物品/金钱）、`.learnings/SUBPLOTS.md`（子剧情）、`.learnings/EMOTIONS.md`（情感发展）

**写入规范**：
- 新角色出场 → 立即追加到 `CHARACTERS.md`
- 角色状态变化（升级/受伤/死亡）→ 立即更新对应条目
- 新地点出现 → 追加到 `LOCATIONS.md`
- 关键情节发生 → 追加到 `PLOT_POINTS.md`
- 伏笔埋设/回收 → 更新 `SUSPENSE.md` 状态
- 世界观规则补充 → 更新 `STORY_BIBLE.md`

## Phase 11：一致性审核

**目标**：检查正文与设定、前文的一致性（剧情/角色/设定层面）

检查维度：核心一致性（剧情/人设/设定前后矛盾） / 格式一致性（对话格式/场景切换/时间线） / 逻辑连贯（动机/因果链/时间线）

> 完整审核维度和评分体系见 `references/quality-check.md`。

## Phase 12：润色与评分

**目标**：对正文做最终润色（语言质量层面），确保质量达标

> Phase 11 负责"一致性"，Phase 12 负责"语言质量"（去AI味/润色/评分）。

### 去AI味检查项（润色必过）

> 详见 `references/anti-ai-detection.md`

核心检查项：①比喻词未超频（每章≤1处功能性比喻） ②直白心理描写≤1处 ③禁语表词汇未超频 ④段落不超过4行 ⑤章尾无总结/升华/预告 ⑥当前文风例外规则已正确应用

### 质量检查

> 完整检查清单见 `references/quality-check.md`

通用速查：开头有钩子 / 中段有推进 / 局势有变化 / 结尾落在变化上 / 没有大段设定说明文 / 对话符合人物身份 / 情绪通过动作落地

水文检测：全章对话无新信息 / 同一情绪写3段以上 / 场景描写超500字不推进剧情 / 连续2章以上无冲突

### 出稿门槛

| 场景 | 最低评分 |
|------|---------|
| 普通日更 | 总分 >= 7.0 |
| 关键高潮章 | 总分 >= 8.0 |
| 卷终章 | 总分 >= 8.2 |
| 发布前精修 | 必跑深度审稿 |

### 记忆写入（润色评分后必执行）

| 文件 | 写入内容 |
|------|---------|
| `SESSION.md` | 更新章节状态为"已润色"、更新评分记录 |
| `.learnings/ERRORS.md` | 质量问题记录（评分低于出稿门槛时） |

## 修改流程

### 大纲修改

1. 修改 `大纲/大纲.md` 和受影响的 `大纲/卷纲_第X卷.md` 2. 评估影响：仅影响未写章节→标记"待返修"；影响已写正文→标记"待重写" 3. 受影响批次细纲解冻→重新走 Phase 8→9 4. 如有"待重写"章节→重新走 Phase 10-12 5. 同步写入记忆

| 级别 | 判断标准 | 处理方式 |
|------|---------|---------|
| 局部微调 | 只改细节，不影响后续走向 | 修改大纲，同步更新细纲 |
| 结构调整 | 改变卷级走向或角色命运 | 修改大纲+卷纲，受影响批次解冻返修 |
| 主线重构 | 核心冲突/金手指/结局变化 | 全部未冻结细纲解冻，从头返修 |

### 细纲修改

1. 目标批次细纲解冻→修改→重新走 Phase 9 审核冻结 2. 已写正文受影响→标记"待重写"→重走 Phase 10-12 3. 同步写入记忆

### 修改后记忆同步

须同步更新：`.learnings/STORY_BIBLE.md`（世界观变更）、`.learnings/CHARACTERS.md`（角色变更）、`.learnings/PLOT_POINTS.md`（情节变更）、`.learnings/SUSPENSE.md`（伏笔变更）、`大纲/细纲迭代记录.md`（返修记录）、`SESSION.md`（进度更新）

## 常见问题 (Troubleshooting)

### AI 不按流程走

**问题**：AI 跳过了某些 Phase 直接生成正文。
**解决**：明确指示"请按 Phase X 继续"，或检查前置依赖矩阵是否满足。

### 记忆文件不同步

**问题**：角色状态/地点/伏笔与正文不一致。
**解决**：手动运行一致性审核（Phase 11），检查 `.learnings/` 中对应文件并更新。

### 细纲冻结后想修改

**问题**：已冻结的细纲发现有问题。
**解决**：走修改流程→解冻目标批次→重新走 Phase 8→9 审核冻结→如已写正文则重走 Phase 10-12。

### 去AI味后文字太干瘪

**问题**：严格执行去AI味规则后，文字变得没有感情。
**解决**：检查是否激活了正确的文风。古风仙侠风允许每章2处意境比喻，轻松日常风允许幽默比喻。见 `references/anti-ai-detection.md` 文风例外说明。

### 长篇写到中后段质量下滑

**问题**：前50章质量不错，后面开始跑偏。
**解决**：启用深度审稿模式（`references/advanced-audit.md`），做24维排查。或启用卡文救援（`references/continuation-engine.md`）。

### 想从半部小说继续

**问题**：已经写了几万字，想让 AI 接着写。
**解决**：先走项目进度检测（见上方表格），判断当前处于哪个 Phase，然后从对应阶段恢复。半部续写先快速补齐 Phase 1-4 设定。

### 擦边描写与去AI味冲突

**问题**：擦边风格要求用心理活动，去AI味禁止心理描写。
**解决**：用生理反应替代心理描写（"他喉结动了一下"而非"他心跳加速"）。擦边风格C（文艺唯美型）为例外，允许在擦边情节中使用比喻。见 `references/anti-ai-detection.md` 优先级规则。

### 想快速生成一章试试

**问题**：不想走完整流程，只想快速看一章效果。
**解决**：指定"快速模式"，AI 会跳过 Phase 1-3 详细设定，直接基于你的题材方向生成细纲和正文。
