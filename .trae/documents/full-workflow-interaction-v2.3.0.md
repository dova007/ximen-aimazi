# V2.3.0 全流程交互决策点优化计划

## 核心目标

将续写流程已有的"交互决策点"模型扩展到整个创作流程（Phase 1-12），实现：
1. 每一步都有用户介入决策的可能性，提供结构化交互选项
2. 无论用户从哪一步切入，都完整走完剩余流程
3. 除非用户明确表示跳过，否则不省略任何步骤

## 当前差距分析

| 维度 | 续写流程（已有） | 主流程（现状） | 目标 |
|------|-----------------|---------------|------|
| 交互模式 | 讨论模式/快速模式 | 无 | 统一两种模式 |
| 决策点数量 | 12个 | 仅Phase4/9有 | 覆盖Phase1-12 |
| 交互格式 | 4要素规范 | 简单确认 | 统一4要素 |
| 切入恢复 | 进度检测表 | 仅续写场景有 | 全流程通用 |
| 选项设计 | 2-4选项+自定义 | 确认/修改 | 全部结构化 |

## 实施步骤

### 步骤1：创建 `references/workflow-interaction.md`

新建全流程交互决策点指南文件，定义 Phase 1-12 的交互决策点。

**设计原则**：
- 每个Phase至少1个决策点，关键Phase有2-3个
- 复用续写流程的4要素交互格式（状态摘要/选项列表/推荐标注/上下文提示）
- 统一两种交互模式（讨论模式/快速模式），与续写流程一致
- 快速模式下的必停点：DP1-2（方案选择）、DP4-1（参数配置）、DP9-1（审核方案选择）、DP10-2（正文审核）

## 决策点详细设计

### DP1-1：创作方向确认

**时机**：用户提供初始方向后，AI补全8维度提示词后
**模式**：讨论模式必停 / 快速模式跳过

**状态摘要**：
"收到你的方向！让我确认一下我的理解，并补充一些你可能没想到的维度。"

**交互内容**：
```
🎯 创作方向确认：

你的方向：{用户原始输入}
我的理解：{AI理解摘要}

8维度补全：
- 题材定位：{补全}
- 世界观方向：{补全}
- 主角人设方向：{补全}
- 核心冲突方向：{补全}
- 爽点设计方向：{补全}
- 节奏规划方向：{补全}
- 配角框架方向：{补全}
- 开篇钩子方向：{补全}

❓ 我的理解和补充方向对吗？
```

**选项**：
| 选项 | 说明 |
|------|------|
| A. 方向正确，继续生成方案 | 进入DP1-2 |
| B. 某些维度需要调整 | 用户指出后逐项调整 |
| C. 方向不太对，让我重新描述 | 重新收集方向 |
| D. 先给我看看参考题材 | 加载genre-frameworks-unified.md展示 |

## 异常处理

| 情况 | 处理方式 |
|------|---------|
| 用户在决策点选择"自定义" | 收集用户描述后调整方案 |
| 用户连续3次选择"不满意" | 建议回退到上一个决策点重新讨论 |
| 用户在快速模式下要求讨论 | 立即切换到讨论模式 |
| 用户跳过决策点 | 记录到 SESSION.md，使用 AI 推荐选项 |
| 用户在决策点提出新需求 | 评估是否需要回退到更早的阶段 |
| 用户从中间切入 | 执行进度检测，从恢复位置开始，走完剩余流程 |
| 用户明确表示跳过某Phase | 记录到 SESSION.md，跳过该Phase所有决策点 |
